## Context

Sprint 4 (chapter portability + course package export) is in-flight on `feat/chapter-portability-export`.
This plan defines Sprint 5: the curriculum design and content creation platform for IIBA East Tennessee Chapter.

**The pain:** Q2 2026 Career Accelerator launches in weeks. ECBA Sessions 2–5 have no slides.
Career Accelerator has 8 session types with zero content authored. The facilitator tool
is useless without content to run.

**The goal:** A content authoring platform that lets the PD lead (and eventually any
board member) create, edit, preview, and publish curriculum for all IIBA ETN program types
— without touching the filesystem or writing code.

---

## Vision

### 10x Check
The 10x version is a **self-service curriculum operating system**: any IIBA ETN board member
opens the Content Studio, selects a session type, enters a topic, and AI generates a complete
first-draft content package (slides, handout, facilitator script, practice questions) in IIBA ETN
style. The PD lead reviews and edits. One-click publish makes it live in the facilitator app.
A content gap radar shows exactly what's missing before Q2 launch. Facilitators see only
published sessions. Style guide violations surface in real time.

### Platonic Ideal (Experience-First)

**The curriculum designer opens the Content Studio and sees:**
- Program map: all sessions color-coded by status (gap / draft / review / published)
- "ECBA Session 2 — Missing slides, handout ready" in red
- "Career Accelerator: 0/8 sessions authored — 38 days to Q2 launch"
- `+ New Session` button that scaffolds everything

**She clicks New Session → Career Accelerator → Resume Lab:**
- Gets a pre-structured template: Pre-work | Lab Instructions | Rubric | Facilitator Notes
- Not a blank page — 80% done before she types a word
- "Generate First Draft" pulls from TrailBlaze case context + ECBA Style Guide
- Real-time warnings: "Slide 3 has 7 bullets (max 5)" | "Missing learning objective"
- "Preview as Facilitator" renders exactly what the facilitator will see

**She publishes:**
- Status changes to Published
- Facilitator app immediately picks it up (filter: only published sessions)
- Program map turns green

## Accepted Scope (Sprint 5)

### Core: `apps/content_studio.py`
New Streamlit app. Separate from `facilitator_ui.py` (different persona: author vs facilitator).

**Content type registry** — defined in `apps/content_types.py`:
```python
CONTENT_TYPES = {
    "ecba_session": {
        "slots": ["slides", "handout", "facilitator_script", "practice_questions"],
        "required": ["slides", "handout"],
    },
    "career_accelerator": {
        "subtypes": [
            "career_mapping", "resume_lab", "mock_interview",
            "case_workshop", "requirements_fundamentals",
            "skills_not_titles_panel", "career_retrospective",
            "hiring_manager_perspective"
        ],
        "slots": ["facilitator_guide", "participant_handout", "pre_work", "rubric"],
        "required": ["facilitator_guide", "participant_handout"],
    },
    "panel_event": {
        "slots": ["moderator_script", "panel_agenda", "panelist_bios",
                  "networking_rotation", "post_event_followup"],
        "required": ["moderator_script", "panel_agenda"],
    },
}
```

**Content status workflow** — YAML frontmatter in every content file:
```yaml
---
status: draft  # draft | published  (Sprint 5; "review" is a future state for Sprint 6+)
content_type: ecba_session
session_id: 2       # ECBA: 1–5 | Career Accelerator: 100–107 (100=career_mapping, 101=resume_lab,
                    # 102=mock_interview, 103=case_workshop, 104=requirements_fundamentals,
                    # 105=skills_not_titles_panel, 106=career_retrospective, 107=hiring_manager_perspective)
                    # Multiple instances of same type: 101_v2, 101_v3, etc.
slot: slides
last_modified: 2026-03-28
---
```

**Status values for Sprint 5:** `draft` (not ready to run) and `published` (live in facilitator app).
`review` is defined in the platonic ideal but deferred to Sprint 6+ when a second reviewer exists.
In Sprint 5, the PD lead both authors and approves: draft → published directly.

**Facilitator app gate** — `apps/shared.py` session loader filters to `status: published` only.
Existing Session 1 files get `status: published` backfilled (they're already live).

### Content Gap Radar
Home screen of Content Studio. For each session in the program:
- What content slots exist (files found)
- What's missing (slots defined in type but no file)
- Status of each slot (draft/review/published)
- Days until Q2 launch (hardcoded date: 2026-04-01 per `q2_mvp_calendar.md`)

Implementation: filesystem scan of `etn/` for frontmatter + cross-reference against
`CONTENT_TYPES` slot definitions. No database needed.

**Caching:** Gap Radar scan is cached with `@st.cache_resource` for 60 seconds per Streamlit
session. After successfully writing to disk (file write confirmed), call `st.cache_resource.clear()`
to invalidate the Gap Radar cache on next render. If the file write fails, do NOT clear the cache
— the radar should reflect the last successful state. This prevents stale radar on active
authoring sessions while maintaining consistency on write failure.
Cold scan of up to 200 files in `etn/` completes in <200ms on a local SSD; caching is for
UX smoothness, not performance necessity.

### AI-Assisted First Draft
"Generate First Draft" button per content slot. Grounding documents:
- `etn/ECBA_CaseStudy/TrailBlaze_MasterContext.md` — case study context
- `etn/ECBA_CaseStudy/Facilitator/ECBA_Style_Guide.md` — style rules
- `etn/ECBA_CaseStudy/Session1_Slides.md` — example output (Session 1 as few-shot example)

Uses `agents/llm_adapter.py`. Prompt template per slot type (slides vs handout vs script differ).
Output is a first draft in the editor — NOT auto-saved. Author reviews, edits, then saves.

Human review gate: **required** before publish. Generated content is placed in the editor as
`draft` status — it is NOT auto-saved. The author reads, edits, then explicitly saves.
Promoting to `published` is a separate manual action (a button: "Publish Session").
There is no auto-publish path.

### Style Guide Enforcer
Runs on every content save (debounced — not keystroke-by-keystroke; triggered on save action).
Rules are **hard-coded** in `apps/content_types.py` for Sprint 5 (dynamic rule extraction from
the style guide markdown is deferred to Sprint 6). All 4 Sprint 5 rules:

```python
STYLE_RULES = [
    StyleRule("max_bullets_per_slide", max_count=5,
              message="Slide {n} has {count} bullets (max 5)"),
    StyleRule("required_learning_objective",
              pattern=r"^## Learning Objective",
              message="Missing '## Learning Objective' header"),
    StyleRule("facilitator_cue_format",
              pattern=r"\[FACILITATOR:",
              message="Facilitator cue on line {n} should use [FACILITATOR: ...] format"),
    StyleRule("slide_count_range", min_count=3, max_count=15,
              message="Session has {count} slides (expected 3–15)"),
]
```

Warnings displayed inline (yellow `st.warning` banner), not hard blocks. Author can dismiss.
No auto-fix in Sprint 5 — warnings only.

### Preview as Facilitator
"Preview" tab in the Content Studio. Renders selected session's slides using `parse_slides()`
from `facilitator_ui.py`. Read-only — no session start, no timer. Shows cues, structure,
and slide sequence exactly as the facilitator will see it.

- `career_mapping_facilitator_guide.md`
- `resume_lab_facilitator_guide.md`
- `mock_interview_facilitator_guide.md`
- `case_workshop_facilitator_guide.md`
- `requirements_fundamentals_facilitator_guide.md`
- `skills_not_titles_panel_facilitator_guide.md`
- `career_retrospective_facilitator_guide.md`
- `hiring_manager_perspective_facilitator_guide.md`

Each pre-populated with the structure appropriate for that session type.

**Template scaffolding — concrete example:**
Clicking `+ New Session → Career Accelerator → Resume Lab` creates:
`etn/career_accelerator/2026-03-28_resume_lab_facilitator_guide.md`
with frontmatter auto-populated:
```yaml
---
status: draft
content_type: career_accelerator
subtype: resume_lab
session_id: 101
slot: facilitator_guide
created: 2026-03-28
last_modified: 2026-03-28
---
```
Filename pattern: `{YYYY-MM-DD}_{subtype}_{slot}.md`. Date prefix enables sorting by creation date.
If a file with that session_id + slot already exists, prompt: "A facilitator guide for Resume Lab
already exists. Create a new version (session_id: 101_v2)?" No silent overwrites.

### CIO/Networking Event Content Type + Panel Mode
New content type: `panel_event` (defined in type registry above).
Templates:
- `etn/templates/panel_event/moderator_script.md`
- `etn/templates/panel_event/panel_agenda.md`
- `etn/templates/panel_event/networking_rotation.md`

**Panel Mode scope note:** This is a **minimal facilitator_ui.py change only** — NOT a new
Content Studio authoring UI. The Content Studio creates and edits panel_event files the same
way as any other content type (textarea editor + save). The only new code is in facilitator_ui.py:
a `mode='panel'` branch in the session renderer that shows agenda + timer + moderator cues
instead of slide rendering. Total new code: ~50 lines in facilitator_ui.py + 3 template files.
Panel Mode does NOT add a new content creation workflow; it adds a new content *display* workflow.

**Timeline impact:** Panel Mode is pulled forward from "Hour 3" to its own sub-sprint.

## Implementation Notes

### Sprint 5 Timeline (human: ~2 weeks / CC: ~3 hours)
- Hour 1: `apps/content_types.py` registry + `apps/content_studio.py` skeleton + Gap Radar
- Hour 1.5: Content status workflow (frontmatter reader/writer) + facilitator filter gate
- Hour 2: 8 Career Accelerator templates + CIO/panel templates + content type registration
- Hour 2.5: AI first-draft (llm_adapter.py integration) + human review gate
- Hour 3: Style guide enforcer + Preview as Facilitator + tests

### Key architectural constraints
1. `apps/content_studio.py` must NOT import from `apps/facilitator_ui.py` directly.
   Shared code (parse_slides, frontmatter utils) moves to `apps/shared.py`.
2. **Frontmatter implementation:** Use the `python-frontmatter` library. Open files with
   `open(path, encoding='utf-8', newline='')` to preserve line endings (CRLF on Windows).
   Idempotence test: write → read → write → `assert open(path,'rb').read() == original_bytes`.
   Add this as a unit test: `tests/test_frontmatter_idempotence.py`.
3. **AI graceful degradation:** `llm_adapter.py` must export:
   `generate_content_draft(content_type: str, slot: str, context: dict) -> str`
   Raises `APIKeyError` (defined in llm_adapter) if `ANTHROPIC_API_KEY` is not set.
   Raises `ContentGenerationError` on API failure or empty response.
   Max context: 16K tokens (Claude claude-haiku-4-5-20251001 or equivalent fast model for drafts).
   On `ContentGenerationError`, preserve the editor's current content and show a retry option —
   do NOT clear the editor on API failure. Concurrent edit assumption: single-user per local
   session; no file locking in Sprint 5 (deferred to hosted deployment phase).
   Token budget monitoring: if grounding documents exceed 9K tokens combined, fall back to
   style guide only (discard case study context). Log a `WARNING` when fallback is triggered.
   Content Studio checks for `APIKeyError` at startup; hides "Generate First Draft" button
   and shows "AI generation not configured (set ANTHROPIC_API_KEY)" if key is absent.
4. Panel Mode in facilitator_ui.py is a new `mode` kwarg to the session renderer, not a
   separate code path. Keeps the session lifecycle consistent.

## Temporal Interrogation

What the implementer needs to know at each stage:

**Hour 1 (foundations):**
- Does `shared.py` already export `parse_slides`? → YES at line ~180. Move it, don't duplicate.
- What's the frontmatter format? → Use `python-frontmatter` library (already in requirements?) or
  hand-roll a simple YAML header parser. Check `requirements.txt` first.
- Where does the Gap Radar get the Q2 launch date? → `docs/q2_mvp_calendar.md` or hardcode `2026-04-01`.

**Hour 1.5 (content status):**
- Will adding `status: published` frontmatter to existing Session 1 files break the facilitator app?
  → Only if `facilitator_ui.py` does a raw file read without frontmatter stripping. Check `parse_slides()`.

**Hour 2 (templates):**
- Should templates be in `etn/templates/` or a separate `templates/` at root?
  → `etn/templates/` keeps it chapter-specific and consistent with `chapter_config.yaml` path conventions.

**Hour 2.5 (AI generation):**
- Check if `llm_adapter.py` supports system + user message format. Prompt format:
  system = (style guide content), user = "Generate a slide deck for ECBA Session {N}: {topic}.
  Use the TrailBlaze case study. Follow the IIBA ETN style guide exactly."
- Graceful degradation: check `APIKeyError` from llm_adapter at startup; hide button + show
  config message. Do NOT raise to user. Do NOT retry on error.
- Token budget: TrailBlaze_MasterContext.md is ~4K tokens; Session1_Slides.md ~2K tokens;
  style guide ~3K tokens. Total grounding context ~9K tokens. Leaves ~7K for output.
  If output is truncated (no closing `---` slide separator), show warning: "Draft may be
  incomplete — check the last slide."

**Hour 3 (style guide enforcer):**
- Rules are **hard-coded** as `STYLE_RULES` list in `apps/content_types.py` (see rule spec above).
  Do NOT parse `ECBA_Style_Guide.md` at runtime in Sprint 5.
- Debounce: use `st.session_state["last_validation_ts"]` + `time.time()` — run validation
  if >2 seconds since last run AND file content has changed.
